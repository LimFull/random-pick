# 계획 구현

`thoughts/shared/plans/`에서 승인된 기술 계획을 구현하는 것이 당신의 임무입니다. 이 계획들은 특정 변경사항과 성공 기준이 포함된 단계들을 포함합니다.

## 시작하기

계획 경로가 주어졌을 때:

- 계획을 완전히 읽고 기존 체크마크(- [x])가 있는지 확인하세요
- 원본 티켓과 계획에서 언급된 모든 파일을 읽으세요
- **파일을 전체적으로 읽으세요** - limit/offset 매개변수를 사용하지 마세요, 완전한 컨텍스트가 필요합니다
- 조각들이 어떻게 맞춰지는지 깊이 생각하세요
- 진행 상황을 추적하기 위해 할 일 목록을 생성하세요
- 수행해야 할 것을 이해했다면 구현을 시작하세요

계획 경로가 제공되지 않으면 요청하세요.

## 구현 철학

계획은 신중하게 설계되지만, 현실은 복잡할 수 있습니다. 당신의 역할은:

- 발견한 것에 적응하면서 계획의 의도를 따르세요
- 다음 단계로 넘어가기 전에 각 단계를 완전히 구현하세요
- 작업이 더 넓은 코드베이스 컨텍스트에서 의미가 있는지 확인하세요
- 섹션을 완료할 때마다 계획의 체크박스를 업데이트하세요

사물이 계획과 정확히 일치하지 않을 때, 왜 그런지 생각하고 명확하게 소통하세요. 계획은 당신의 가이드이지만, 당신의 판단도 중요합니다.

불일치를 만나면:

- 중단하고 계획을 따를 수 없는 이유를 깊이 생각하세요
- 문제를 명확하게 제시하세요:

  ```
  [N]단계의 문제:
  예상: [계획이 말하는 것]
  발견: [실제 상황]
  이것이 중요한 이유: [설명]

  어떻게 진행해야 할까요?
  ```

## 검증 접근 방식

단계를 구현한 후:

- 성공 기준 검사를 실행하세요 (보통 `npm run build --workspaces`, `npm run test --workspace=backend`, `npm run lint --workspaces`)
- 진행하기 전에 문제를 수정하세요
- 계획과 할 일 목록 모두에서 진행 상황을 업데이트하세요
- Edit을 사용하여 계획 파일 자체에서 완료된 항목을 체크하세요
- **인간의 검증을 위해 일시정지**: 단계의 모든 자동화된 검증을 완료한 후, 일시정지하고 인간에게 단계가 수동 테스트를 위해 준비되었다고 알리세요. 이 형식을 사용하세요:

  ```
  [N]단계 완료 - 수동 검증 준비됨

  자동화된 검증 통과:
  - [ ] 백엔드가 성공적으로 빌드됨: `npm run build --workspace=backend`
  - [ ] 프론트엔드가 성공적으로 빌드됨: `npm run build --workspace=frontend`
  - [ ] 백엔드 테스트 통과: `npm run test --workspace=backend`
  - [ ] 린팅 통과: `npm run lint --workspaces`

  계획에 나열된 수동 검증 단계를 수행해주세요:
  - [ ] localhost:3002에서 프론트엔드 기능 테스트
  - [ ] localhost:3001에서 백엔드 API 엔드포인트 테스트
  - [ ] 프론트엔드와 백엔드 간 통합 확인
  - [계획의 다른 수동 검증 항목들 나열]

  수동 테스트가 완료되면 [N+1]단계로 진행할 수 있도록 알려주세요.
  ```

여러 단계를 연속으로 실행하라는 지시를 받으면, 마지막 단계까지 일시정지를 건너뛰세요. 그렇지 않으면 한 단계만 수행한다고 가정하세요.

사용자가 확인할 때까지 수동 테스트 단계의 항목을 체크하지 마세요.

## 막힐 때

예상대로 작동하지 않을 때:

- 먼저 관련된 모든 코드를 읽고 이해했는지 확인하세요
- 계획이 작성된 이후 코드베이스가 발전했는지 고려하세요
- TypeORM 엔티티, NestJS 모듈, 또는 React 컴포넌트가 업데이트가 필요한지 확인하세요
- 불일치를 명확하게 제시하고 지침을 요청하세요

하위 작업을 절약해서 사용하세요 - 주로 다음과 같은 대상 디버깅이나 익숙하지 않은 영역 탐색에 사용하세요:

- 기존 스케줄러나 notion 모듈 패턴 이해
- TypeORM 엔티티 관계 조사
- 프론트엔드의 React 컴포넌트 구조 탐색

## 작업 재개

계획에 기존 체크마크가 있는 경우:

- 완료된 작업이 완료되었다고 신뢰하세요
- 첫 번째 체크되지 않은 항목부터 시작하세요
- 뭔가 이상해 보이는 경우에만 이전 작업을 검증하세요

기억하세요: 당신은 단순히 체크박스를 확인하는 것이 아니라 솔루션을 구현하고 있습니다. 최종 목표를 염두에 두고 전진 동력을 유지하세요. 변경사항이 전체 가사 작업 스케줄러 시스템에 어떻게 맞는지 고려하고, 스케줄러와 notion 모듈의 기존 패턴과 일관성을 유지하세요.
