# 돌림판 당첨자 선정 웹 애플리케이션 요구사항 문서

## 프로젝트 개요

React를 이용한 SPA(Single Page Application) 웹 애플리케이션으로, 참가 인원들의 이름을 입력하여 돌림판을 돌려 랜덤으로 당첨자를 선정하는 서비스를 제공합니다.

## 기술 스택

- **프레임워크**: React
- **배포 플랫폼**: GitHub Pages
- **애플리케이션 타입**: PWA (Progressive Web App)
- **물리엔진**: 물리엔진 라이브러리 (예: Matter.js, Cannon.js 등)

## 주요 요구사항

### 1. 배포 환경

- GitHub Pages에 배포되는 React SPA 프로젝트
- `main` 브랜치에 푸시하는 것만으로 자동으로 GitHub Pages에 배포되는 환경 구성
- GitHub Actions 또는 GitHub Pages 설정을 통한 자동 배포 파이프라인 구축

### 2. PWA 기능

- Progressive Web App으로 동작해야 함
- 오프라인에서도 동작 가능
- 모바일 기기에 설치 가능
- Service Worker 구현
- Web App Manifest 파일 제공

### 3. 핵심 기능: 돌림판 당첨자 선정

- 참가 인원들의 이름을 입력받아 돌림판을 구성
- 돌림판을 돌려 랜덤으로 당첨자 1명을 선정
- 당첨자 선정 결과를 화면에 표시

### 4. 당첨자 선정 로직

- **가장 중요하고 투명하게 공개되어야 하는 부분**
- 주석과 함께 깔끔한 함수로 정리
- 별도의 파일에 작성 (예: `src/utils/winnerSelection.js` 또는 `src/logic/winnerSelection.ts`)
- 랜덤 알고리즘이 공정하고 투명하게 구현되어야 함
- 시드값이나 예측 가능한 요소가 없어야 함

### 5. 데이터 저장

- 참가 인원 목록은 로컬 스토리지(Local Storage)에 저장
- 다음번 실행 시에도 저장된 참가 인원 목록이 그대로 유지되어야 함
- 브라우저를 닫았다가 다시 열어도 데이터가 유지되어야 함

### 6. 참가 인원 관리 UI/UX

- 참가 인원을 쉽게 추가할 수 있는 기능
- 참가 인원을 쉽게 제거할 수 있는 기능
- 직관적이고 사용하기 편한 인터페이스 제공
- 입력 폼, 목록 표시, 삭제 버튼 등 기본적인 CRUD 기능 구현

### 7. 돌림판 구현

- 물리엔진 라이브러리를 활용하여 리얼하게 돌아가는 돌림판 구현
- 자연스러운 회전 애니메이션
- 관성과 마찰 등 물리적 효과 적용
- 시각적으로 매력적인 디자인

### 8. 문서화

- `README.md` 파일에 당첨자 선정 로직의 구현 위치를 명시
- 프로젝트 설정 방법, 실행 방법, 배포 방법 등 포함
- 당첨자 선정 알고리즘에 대한 설명 포함

### 9. 당첨자 공개 이펙트

- 돌림판이 완전히 멈춘 후 폭죽이 터지는 이펙트 표시
- 당첨자 이름을 화면에 크게 표시
- 축하 애니메이션 및 효과 제공
- 사용자 경험을 향상시키는 시각적 피드백

## 구현 우선순위

1. **Phase 1**: 기본 React 프로젝트 설정 및 GitHub Pages 배포 환경 구성
2. **Phase 2**: 참가 인원 관리 기능 (추가/제거) 및 로컬 스토리지 연동
3. **Phase 3**: 당첨자 선정 로직 구현 및 별도 파일 분리
4. **Phase 4**: 물리엔진을 활용한 돌림판 구현
5. **Phase 5**: 폭죽 이펙트 및 당첨자 공개 기능
6. **Phase 6**: PWA 기능 구현 (Service Worker, Manifest)
7. **Phase 7**: UI/UX 개선 및 최종 테스트

## 참고사항

- 당첨자 선정 로직은 프로젝트의 핵심이므로 코드 리뷰 시 가장 중요한 검토 대상
- 물리엔진 라이브러리는 프로젝트 성격에 맞는 경량 라이브러리 선택 권장
- GitHub Pages 배포를 위한 빌드 설정 및 경로 설정 주의 필요
